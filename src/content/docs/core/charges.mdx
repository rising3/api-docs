The Charge object represents a single attempt to move money into your Stripe account. PaymentIntent confirmation is the most common way to create Charges, but transferring money to a different Stripe account through Connect also creates Charges. Some legacy payment flows create Charges directly, which is not recommended for new integrations.

## The Charges Object

### Attributes

<Attribute text="id" type="string">
Unique identifier for the object.
</Attribute>
<Attribute text="amount" type="integer">
Amount intended to be collected by this payment. A positive integer representing how much to charge in the smallest currency unit (e.g., 100 cents to charge $1.00 or 100 to charge ¥100, a zero-decimal currency). The minimum amount is $0.50 US or equivalent in charge currency. The amount value supports up to eight digits (e.g., a value of 99999999 for a USD charge of $999,999.99).
</Attribute>
<Attribute text="balance_transaction" type="nullable string">
ID of the balance transaction that describes the impact of this charge on your account balance (not including refunds or disputes).
</Attribute>
<Attribute text="billing_details" type="object">
Billing information associated with the payment method at the time of the transaction.
  <Collapse>
    <Attribute text="billing_details.address" type="nullable object">
    Billing address.
    </Attribute>
    <Collapse>
      <Attribute text="billing_details.address.city" type="nullable string">
      City, district, suburb, town, or village.
      </Attribute>
      <Attribute text="billing_details.address.country" type="nullable string">
      Two-letter country code (ISO 3166-1 alpha-2).
      </Attribute>
    </Collapse>
    <Attribute text="billing_details.email" type="nullable string">
    Email address.
    </Attribute>
    <Attribute text="billing_details.name" type="nullable string">
    Full name.
    </Attribute>
    <Attribute text="billing_details.phone" type="nullable string">
    Billing phone number (including extension).
    </Attribute>
  </Collapse>
</Attribute>
<Attribute text="currency" type="enum">
Three-letter ISO currency code, in lowercase. Must be a supported currency.
</Attribute>

## Create a charge
<Badge text="Deprecated" variant="danger" />
This method is no longer recommended—use the Payment Intents API to initiate a new payment instead. Confirmation of the PaymentIntent creates the Charge object used to request payment.

### Parameters
<Attribute text="amount" type="integer" required="true">
Amount intended to be collected by this payment. A positive integer representing how much to charge in the smallest currency unit (e.g., 100 cents to charge $1.00 or 100 to charge ¥100, a zero-decimal currency). The minimum amount is $0.50 US or equivalent in charge currency. The amount value supports up to eight digits (e.g., a value of 99999999 for a USD charge of $999,999.99).
</Attribute>
<Attribute text="currency" type="enum" required="true">
Three-letter ISO currency code, in lowercase. Must be a supported currency.
</Attribute>
<Attribute text="customer" type="string">
The ID of an existing customer that will be charged in this request.
</Attribute>
<Attribute text="description" type="string">
An arbitrary string which you can attach to a `Charge` object. It is displayed when in the web interface alongside the charge. Note that if you use Stripe to send automatic email receipts to your customers, your receipt emails will include the `description` of the charge(s) that they are describing.
</Attribute>
<Attribute text="metadata" type="object">
Set of key-value pairs that you can attach to an object. This can be useful for storing additional information about the object in a structured format. Individual keys can be unset by posting an empty value to them. All keys can be unset by posting an empty value to `metadata`.
</Attribute>
<Attribute text="receipt_email" type="string">
The email address to which this charge’s receipt will be sent. The receipt will not be sent until the charge is paid, and no receipts will be sent for test mode charges. If this charge is for a Customer, the email address specified here will override the customer’s email address. If `receipt_email` is specified for a charge in live mode, a receipt will be sent regardless of your email settings.
</Attribute>

### Returns
Returns a balance object for the account that was authenticated in the request.
